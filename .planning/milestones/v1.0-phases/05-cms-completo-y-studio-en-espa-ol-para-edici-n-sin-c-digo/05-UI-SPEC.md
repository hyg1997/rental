# Phase 5 — UI Design Contract

> Visual and interaction contract for Phase 5: CMS completo y Studio en español para edición sin código.
> Generated by gsd-ui-researcher.

---

## Scope Note

Phase 5 introduces no new public-facing pages. The frontend HTML output remains visually identical to what Phases 1-3 produced. The design contract for this phase governs two surfaces:

1. **Front-end component API:** Components that previously had hardcoded content (ServicesSection, ValuesSection, MetricsSection, NosotrosPage) must accept props and render empty-state UI when Sanity returns no data.
2. **Sanity Studio UX:** The Studio chrome, sidebar, and field authoring experience for the non-technical client editor.

All tokens below come from the existing codebase — no new tokens are introduced in this phase.

## Empty State Design

When Sanity returns no data (empty array or null — e.g., siteSettings document not yet populated), each section must render a graceful fallback, not a broken layout.

| Component | Empty State Behavior |
|-----------|----------------------|
| ServicesSection | Section renders with heading + body copy. Grid area hidden (no items = no grid). No error icon. |
| ValuesSection | Section renders with heading only. Grid hidden. |
| MetricsSection | Section renders with heading only. No counters shown. |
| NosotrosPage | Page renders h1 + fallback paragraph. Values grid hidden if nosotrosValores is empty. |

**Empty state is silent** — no "Contenido no disponible" message visible to the public visitor. The section heading remains visible to preserve page structure. This is intentional: content will be populated before launch.

## Sanity Studio UX Contract

This section governs the Studio authoring experience — not the public site.

### Studio Language

All Studio chrome (menus, buttons, validation messages, toast notifications) must render in Spanish via the `@sanity/locale-es-es` plugin.

Plugin import: `import { esESLocale } from '@sanity/locale-es-es'`
Plugin registration: Added to `plugins` array in `sanity.config.ts`.

### Sidebar Structure (Structure Builder)

The sidebar must present content in this exact order with these exact Spanish labels:

```
Configuración General      ← siteSettings singleton (single document, no list)
─────────────────────────
Equipos                    ← equipo document list
Banners Hero               ← banner document list
Artículos del Blog         ← post document list
Páginas                    ← pagina document list
─────────────────────────
Reclamos Recibidos         ← reclamo document list (read in practice; no write CTA)
```

Singletons (siteSettings) render as a direct edit pane — no "New document" button, no list view.

### siteSettings Tab Groups

The siteSettings singleton form must use these tabs (fieldGroups) in this order:

| Tab name | `name` value | Fields grouped |
|----------|-------------|----------------|
| Información General | general | siteName, logo |
| Página de Inicio | homepage | servicios[], valores[], metricas[] |
| Página Nosotros | nosotros | nosotrosHistoria, nosotrosMision, nosotrosVision, nosotrosValores[] |
| Contacto y Redes | contacto | whatsappNumber, email, address, socialLinks[] |
| Menú y Footer | navegacion | navLinks[], footerText |

### Icon Field Constraint

All `icono` fields (in servicios, valores, nosotrosValores array members) must use `options.list` with display titles in Spanish. Free-text input for icon names is forbidden — this prevents typo-induced broken icons.

Approved icon sets per field type:
- **Servicios icons:** wrench, check-circle, shield, settings, zap (with Spanish titles)
- **Valores icons (homepage):** award, clock, users, shield, check-circle
- **Nosotros valores icons:** check-circle, shield, users

### Field Descriptions

Every user-visible field must have a `description` in plain Spanish explaining:
- What the field controls on the website
- Any format constraints (e.g., "Número completo con código de país. Ejemplo: 51987654321")
- Character limits or content expectations

No field may have a `description` in English.

## Copywriting Contract

| Element | Copy | Notes |
|---------|------|-------|
| Primary CTA (Studio) | "Guardar y publicar" | shadcn default — Studio publish button already labeled by locale plugin |
| Empty state heading (front-end) | (section heading only, no extra copy) | Silent empty state — see Empty State Design above |
| Empty state body | (hidden — no body copy rendered) | Silent empty state |

| Studio toast — publish success | "Documento publicado" | Auto-provided by locale plugin |
